  git命令流远端推送 2021/7/10 18:28
  介绍了用git命令推送到远端（GitHub）上的命令流。

首先分两种情况，第一种就是首次安装好git后第一次推送，第二种就是第二次之后的重复推送。

第一次推送的话，命令流如下： 

            git init                                                                                                    #初始化，在项目文件下会自动创建.git和.idea文件夹
            git add .                                                                                                   #打包目录下所有文件，准备推送
            git commit -m "..."                                                                                         #这里填写关于版本的介绍内容，也可以随便填
            git branch -M main                                                                                          #切换到main主分支，git默认是master，20年10月份开始使用main
            git status                                                                                                  #查看状态
            git remote add origin https://github.com/...                                                                #链接远端仓库，以后也会默认用这个地址
            git push -u origin main                                                                                     #推送到远端，有时是master，只需把main换成master即可

第二次及以后的重复推送，命令流如下： 

            git add .                                                                                                   #打包文件
            git commit -m "..."                                                                                         #这里填写关于版本的介绍内容，也可以随便填
            git status                                                                                                  #查看状态
            git push                                                                                                    #推到远端
